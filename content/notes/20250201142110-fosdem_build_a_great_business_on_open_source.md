1.  actually makes money

## Model #1: Services for open source projects {#model-1-services-for-open-source-projects}

100% Open Source Software
example: [Percona](https://www.percona.com/)


### Education {#education}

Teach people how to use your software, courses, classes etc.


### Feature Development {#feature-development}

Add features to your software


### Consulting {#consulting}

Help businesses set up your project in their pipeline


### Enterprise Support {#enterprise-support}

**This can have 80% Gross Margin!!**
This is basically insurance for a company


## Model #2: Open Core {#model-2-open-core}

Open Source Software Core + Proprietary Extensions
**This can have 80% Gross Margin!!**
You provide a community version and a pro version
example: [dbeaver](https://dbeaver.io/)


### Extension that can be useful for companies like: {#extension-that-can-be-useful-for-companies-like}

Authentication
Authorization
Clustering
HA/DR
Analytics
Performance Monitoring
Proprietary Libraries
Integration with Proprietary Software


## Model #3: Cloud services for open source {#model-3-cloud-services-for-open-source}

100% Open Source Software + Proprietary Cloud Management
People can self host, you offer a ready cloud service
**50%-80% Gross Margin** It's very sticky, you are likely to keep customers
example: [Altinity](https://altinity.com/)


### You could add: {#you-could-add}

Automation
Upgrade
Performance Management
Monitoring
Security
Support
